# CodeGuard 团队 Review 规则（示例）

分析 PR 时优先检查以下规范，并在发现违反时提高风险等级、标注 blocks_merge：

## 安全
- 禁止硬编码密钥、Token、密码
- 外部输入必须校验（Controller / API 入口）
- SQL 拼接需参数化

## 并发与一致性
- 订单/支付/库存状态变更需考虑幂等与并发
- 分布式场景注意事务边界与补偿逻辑

## 可维护性
- 避免过大函数与重复逻辑
- 公共 API 变更需有测试覆盖说明

## 性能
- 循环内避免 N+1 查询与阻塞 IO
- 大数据量操作需分页或批处理

若 diff 中无足够证据，不要强行套用规则，应标注「需人工确认」。
